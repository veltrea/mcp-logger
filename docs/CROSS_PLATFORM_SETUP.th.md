# การตั้งค่าข้ามแพลตฟอร์ม (โดยใช้ Google Antigravity)

เอกสารนี้อธิบายวิธีการตั้งค่าสภาพแวดล้อมการทำงานของเครื่องมือและเซิร์ฟเวอร์ MCP ในสภาพแวดล้อมต่างๆ (Windows, macOS, Linux) โดยเน้นความเป็นส่วนตัวและการรักษาความปลอดภัย

## 1. นโยบายความเป็นส่วนตัวและเส้นทาง
เพื่อรักษาความปลอดภัยระดับสูง โปรเจกต์นี้จะแยกชื่อผู้ใช้สาธารณะออกจากชื่อบัญชีของผู้ใช้บนเครื่องคอมพิวเตอร์

> [!IMPORTANT]
> ในเอกสารและเนื้อหาที่สร้างโดย AI เส้นทางสาธารณะจะถูกแทนที่ด้วยตัวแทนดังต่อไปนี้:
> - **ชื่อผู้ใช้บนเครื่อง**: `[LOCAL_USER]` (แทนที่ `primalcolors`)
> - **เส้นทางโปรเจกต์**: `/Users/[LOCAL_USER]/DEV/rust-logger`

## 2. การตั้งค่าบน macOS
macOS เป็นสภาพแวดล้อมหลักสำหรับการรัน Claude Desktop และ Google Antigravity

- **การติดตั้ง**: ใช้ `brew` เพื่อจัดการการอ้างอิง
- **เส้นทางการตั้งค่า MCP**: `~/Library/Application Support/Claude/claude_desktop_config.json`

## 3. การตั้งค่าบน Windows (Remote-Server / Local-Server)
ใช้สำหรับการรันเซิร์ฟเวอร์ Syslog และการตรวจสอบการทำงานของ Windows เฉพาะทาง

- **Syslog**: ใช้ `vlt-syslogd` เป็นบริการพื้นฐาน
- **การเข้ารหัส**: ใช้ UTF-8 เป็นค่าเริ่มต้น สำหรับไฟล์แบตช์ดั้งเดิมอาจใช้ Shift-JIS (CP932)
- **เส้นทางการตั้งค่า MCP**: `%APPDATA%\Claude\claude_desktop_config.json`

## 4. การตั้งค่าบน Linux (Ubuntu / Debian / AlmaLinux)
ใช้สำหรับการทดสอบการรวมระบบและการบันทึก Syslog ระยะไกล

- **WOL (Wake-On-LAN)**: ใช้คำสั่ง `wakeonlan [MAC_ADDRESS]` เพื่อปลุกเครื่องระยะไกล
- **ไดเรกทอรีทำงาน**: `~/DEV` (ซิงค์กับคลาวด์)

## 5. การแจ้งเตือนด้วยเสียง (Speak-MCP)
สำหรับงานที่ใช้เวลานาน คุณสามารถใช้ `speak-mcp` เพื่อแจ้งเตือนเมื่อเสร็จสิ้น
- **ตัวอย่าง**: `speak "กระบวนการติดตั้งเสร็จสมบูรณ์แล้ว"`
