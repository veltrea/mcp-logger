# الإعداد مع LM Studio (ذكاء اصطناعي محلي)

يدعم LM Studio بروتوكول MCP، مما يتيح لك توصيل نماذج الذكاء الاصطناعي المحلية الخاصة بك مباشرة بـ `mcp-logger`.

## 1. المتطلبات الأساسية
- تثبيت أحدث إصدار من LM Studio.
- الحصول على ملف `mcp-logger` الثنائي (عن طريق التجميع أو التنزيل).

## 2. خطوات الإعداد
1. افتح LM Studio.
2. انتقل إلى قسم **My Models / MCP** (أو أيقونة القابس).
3. انقر على "Add MCP Server" أو قم بتحرير ملف `mcp_config.json`.
4. أدخل التكوين التالي:

```json
{
  "mcpServers": {
    "mcp-logger": {
      "command": "/المسار/إلى/mcp-logger",
      "args": ["run-mcp"],
      "env": {}
    }
  }
}
```

## 3. قواعد هامة
- **المسار المطلق**: يتطلب LM Studio مساراً كاملاً للملف الثنائي.
- **الوضع غير التفاعلي**: يجب تشغيل خادم MCP في وضع `run-mcp` للتواصل عبر JSON-RPC دون تدخل من الرسائل العادية.
- **أذونات الشبكة**: تأكد من أن جدار الحماية يسمح للتطبيق بإرسال حزم UDP إلى عنوان IP الخاص بـ Syslog.

## 4. التحليل متعدد البايت
يمكن لـ LM Studio عادةً التعامل مع بيانات UTF-8 جيداً، ولكن لضمان صحة السجلات المرسلة إلى Syslog (خاصة العربية)، لا تنسَ تحديد `encoding: "utf-8"` في استدعاء الأداة.
