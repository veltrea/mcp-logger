# Кроссплатформенная настройка (с использованием Google Antigravity)

В данном документе описано, как настроить среду инструментов и серверов MCP в различных операционных системах (Windows, macOS, Linux), с акцентом на конфиденциальность и безопасность.

## 1. Политика конфиденциальности и пути
Для обеспечения высокого уровня безопасности данный проект отделяет публичные имена пользователей от учетных записей на локальных компьютерах.

> [!IMPORTANT]
> В документах и контенте, созданном ИИ, публичные пути будут заменены следующими заполнителями:
> - **Имя локального пользователя**: `[LOCAL_USER]`
> - **Путь проекта**: `/Users/[LOCAL_USER]/DEV/rust-logger`

## 2. Настройка в macOS
macOS является основной средой для запуска Claude Desktop и Google Antigravity.

- **Установка**: Используйте `brew` для управления зависимостями.
- **Путь конфигурации MCP**: `~/Library/Application Support/Claude/claude_desktop_config.json`

## 3. Настройка в Windows (Remote-Server / Local-Server)
Используется для запуска серверов Syslog и проверки специфических задач Windows.

- **Syslog**: Используйте `vlt-syslogd` в качестве основной службы.
- **Кодировка**: Используйте UTF-8 по умолчанию.
- **Путь конфигурации MCP**: `%APPDATA%\Claude\claude_desktop_config.json`

## 4. Настройка в Linux (Ubuntu / Debian / AlmaLinux)
Используется для тестирования системной интеграции и удаленного логирования Syslog.

- **WOL (Wake-On-LAN)**: Используйте команду `wakeonlan [MAC_ADDRESS]` для удаленного пробуждения машин.
- **Рабочий каталог**: `~/DEV`

## 5. Голосовые уведомления (Speak-MCP)
Для длительных задач вы можете использовать `speak-mcp` для уведомления о завершении.
- **Пример**: `speak "Процесс установки завершен"`
