# MCP लॉगर - विस्तृत मैनुअल

## 1. परिचय
`mcp-logger` एक MCP (मॉडल संदर्भ प्रोटोकॉल) सर्वर है जिसे AI सहायकों और Syslog बुनियादी ढांचे के बीच जोड़ने के लिए डिज़ाइन किया गया है। यह AI को लॉगिंग और स्थिति पर नज़र रखने के लिए Syslog का उपयोग करने की अनुमति देता है।

## 2. कॉन्फ़िगरेशन

### क्लॉड डेस्कटॉप (Claude Desktop)
इसे अपने `%APPDATA%\Claude\claude_desktop_config.json` (Windows) या `~/Library/Application Support/Claude/claude_desktop_config.json` (macOS) में जोड़ें।

```json
{
  "mcpServers": {
    "mcp-logger": {
      "command": "/path/to/mcp-logger",
      "args": ["run-mcp"]
    }
  }
}
```

## 3. टूल संदर्भ

### `send-syslog`
Syslog सर्वर पर संदेश भेजता है।

| पैरामीटर | प्रकार | डिफ़ॉल्ट | विवरण |
| :--- | :--- | :--- | :--- |
| `server` | string | (अनिवार्य) | गंतव्य सर्वर का IP या होस्टनाम। |
| `port` | number | 514 | UDP पोर्ट नंबर। |
| `facility` | string | user | Syslog फैसिलिटी (kern, user, mail आदि) |
| `severity` | string | info | Syslog गंभीरता (emerg, alert, crit, err, warning, notice, info, debug) |
| `tag` | string | mcp-syslog | प्रोग्राम का नाम/टैग। |
| `encoding` | string | utf-8 | कैरेक्टर एन्कोडिंग। |
| `message` | string | (अनिवार्य) | लॉग की सामग्री। |

## 4. मल्टी-बाइट कैरेक्टर गाइड (गैर-ASCII भाषाओं के लिए)
यदि आप हिंदी या अन्य मल्टी-बाइट भाषाओं का उपयोग कर रहे हैं, तो कृपया इन दिशा-निर्देशों का पालन करें:

### अनुशंसा: UTF-8 का उपयोग करें
`mcp-logger` को UTF-8 के लिए अनुकूलित किया गया है। यह स्वचालित रूप से पता लगाता है कि संदेश में गैर-ASCII कैरेक्टर हैं या नहीं और यदि आवश्यक हो तो **UTF-8 BOM (बाइट ऑर्डर मार्क)** प्रदान करता है।

### Syslog सर्वर के साथ तालमेल
कई विंडोज-आधारित Syslog सर्वर (जैसे `vlt-syslogd`) या आधुनिक Linux `rsyslog` सेटअप UTF-8 को संभाल सकते हैं यदि उन्हें सही ढंग से पहचाना जाए। BOM सुनिश्चित करता है कि ये टूल लॉग को स्थानीय विरासत एन्कोडिंग (जैसे CP932) के रूप में गलत नहीं समझेंगे।

## 5. समस्या निवारण
- **AI टूल को नहीं पहचान रहा है**: सुनिश्चित करें कि कॉन्फ़िगरेशन फ़ाइल में पथ पूर्ण (absolute path) है।
- **लॉग नहीं पहुँच रहे हैं**: जांचें कि क्या गंतव्य सर्वर के फ़ायरवॉल पर UDP पोर्ट 514 खुला है।
- **कैरेक्टर त्रुटि**: सुनिश्चित करें कि `encoding` पैरामीटर `utf-8` पर सेट है।
