## अवलोकन
यह प्रोजेक्ट एक MCP (मॉडल संदर्भ प्रोटोकॉल) सर्वर प्रदान करता है जो AI सहायकों (जैसे क्लॉड, जेमिनी आदि) को लॉगिंग और निगरानी के लिए Syslog बुनियादी ढांचे का उपयोग करने में सक्षम बनाता है।
यह स्वयं एक "Syslog सर्वर" नहीं है, बल्कि AI के लिए एक "क्लाइंट कार्यक्षमता" प्रदान करता है ताकि वह मौजूदा Syslog सर्वर (जैसे `rsyslog`, `syslog-ng` या `vlt-syslogd`) को लॉग भेज सके।

यह प्रोजेक्ट `rust-logger` (इंसानों के लिए एक CLI लॉगर) के साथ मुख्य लॉजिक साझा करता है, जबकि AI इंटरैक्शन के लिए अनुकूलित इंटरफ़ेस प्रदान करता है।

## प्रमुख विशेषताएं
- **Syslog संदेश भेजें**: AI सर्वर, पोर्ट, फैसिलिटी (facility), गंभीरता (severity), टैग, एन्कोडिंग और संदेश निर्दिष्ट करके Syslog भेज सकता है।
- **मल्टी-बाइट कैरेक्टर सपोर्ट (BOM के साथ UTF-8)**: विशेष रूप से गैर-ASCII कैरेक्टर (हिंदी, जापानी, चीनी, कोरियाई आदि) को सही ढंग से संभालने के लिए डिज़ाइन किया गया है ताकि AI विश्लेषण के दौरान कोई फ़ॉन्ट त्रुटि न हो।
- **CLI-MCP डुअल डिज़ाइन**: इंसानों के लिए स्टैंडअलोन CLI टूल और AI के लिए MCP सर्वर दोनों के रूप में कार्य करता है।

## स्थापना
रस्ट (Rust) वातावरण की आवश्यकता है।

```bash
cargo build --release
```

## उपयोग कैसे करें (CLI / MCP डुअल मोड)

यह टूल "इंसानों के लिए CLI" और "AI के लिए MCP सर्वर" को जोड़ता है।

### 1. मानव CLI मोड
आप सीधे अपने टर्मिनल से लॉग भेज सकते हैं।

```bash
# बुनियादी संदेश भेजना
mcp-logger send 192.168.1.100 "CLI से नमस्कार"

# विकल्पों के साथ भेजना
mcp-logger send 192.168.1.100 "त्रुटि हुई" --severity error --tag my-app
```

टर्मिनल में बिना किसी तर्क के `mcp-logger` चलाने पर **MCP सेटअप गाइड** दिखाई देगी।

### 2. AI सहायक मोड (MCP)
अपनी `claude_desktop_config.json` या समकक्ष कॉन्फ़िगरेशन फ़ाइल में निम्नलिखित सामग्री जोड़ें।

```json
{
  "mcpServers": {
    "mcp-logger": {
      "command": "/path/to/mcp-logger",
      "args": ["run-mcp"]
    }
  }
}
```

## MCP टूल (Tools)
AI के लिए उपलब्ध विशेषताएं।

### `send-syslog`
निर्दिष्ट सर्वर पर Syslog संदेश भेजता है।

**तर्क (Arguments):**
- `server` (string, अनिवार्य): Syslog सर्वर का होस्टनाम या IP।
- `port` (number): पोर्ट नंबर (डिफ़ॉल्ट: `514`)।
- `facility` (string): फैसिलिटी (जैसे `user`, `local0`) डिफ़ॉล์ट `user` है।
- `severity` (string): गंभीरता (जैसे `info`, `error`) डिफ़ॉल्ट `info` है।
- `tag` (string): प्रोग्राम का नाम (टैग) डिफ़ॉल्ट `mcp-syslog` है।
- `encoding` (string): एन्कोडिंग (जैसे `utf-8`) डिफ़ॉल्ट `utf-8` है।
- `message` (string, अनिवार्य): लॉग संदेश।

## महत्वपूर्ण: मल्टी-बाइट कैरेक्टर हैंडलिंग
हिंदी जैसी भाषाओं के लिए, AI को `utf-8` का उपयोग करने का निर्देश दिया जाता है। सर्वर स्वचालित रूप से **UTF-8 BOM** संलग्न करेगा ताकि Syslog सर्वर और बाद के विश्लेषण उपकरण एन्कोडिंग की सही पहचान कर सकें।
